# Part A — API Design

## А.1 Муу API-н шинжилгээ

### Олдсон 6 алдаа

**Алдаа 1 — Нэршил: товчилсон нэр**
`usr_mgr` → `UserService` / `UserRepository`
Нэр нь класс юу хийдгийг тодорхой харуулах ёстой.

**Алдаа 2 — Мэдээллийн далдлалт: public талбарууд**
`public db_conn`, `public users_arr` — дотоод хэрэгжилтийн дэлгэрэнгүй
мэдээлэл гадагш ил байна. Эдгээр нь `private` байх ёстой.

**Алдаа 3 — Boolean/flag trap: do_user_op(flag)**
`flag: 0=create, 1=update, 2=delete, 3=restore` — нэг метод 4 өөр
ажил хийнэ. Энэ нь `createUser`, `updateUser`, `deleteUser`,
`restoreUser` гэсэн тусдаа методуудаар солигдох ёстой.

**Алдаа 4 — Нэршил: товчилсон, тодорхойгүй нэр**
`get_u`, `find` — параметрийн нэр болон буцаах утгын тухай ямар ч
мэдээлэл байхгүй. `findById`, `findByEmail`, `search` нэрс нь
утгаа тодорхой илэрхийлнэ.

**Алдаа 5 — Ойлголтын жин: any төрөл**
`obj: any`, `flag: number` — төрлийн аюулгүй байдал алга.
`CreateUserInput`, `UpdateUserInput` интерфейс ашиглан оролтыг
тодорхой болгосон.

**Алдаа 6 — Алдаа зохицуулалт: утгаар буцаана**
`'ERR_404'` гэсэн string буцаана — алдааг буцаах утгаар илэрхийлэх
нь алдааны шалтгааныг нуудаг. `UserNotFoundError` exception шиддэг
болгосон.

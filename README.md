## Ажлын товч тойм
А хэсэгт муу API-г шинжлэн 6 алдааг олж засварлав. LRU, LFU, TTL
гурван хэрэгжилттэй кэш сан зохион бүтээж CacheFactory паттернаар
concrete классыг нуув. 19 unit тест бичиж бүгдийг давлаа.

Б хэсэгт Express.js дээр номын сангийн REST API хэрэгжүүлэв.
/books, /members, /loans, /reservations нөөцүүд, RFC 7807 алдааны
формат, OpenAPI 3.0 тодорхойлолт, Postman 15 тест цуглуулга багтсан.

## Тулгарсан бэрхшээл
- partB хавтас partA доор буруу үүссэн — зөв газар зөөж засав
- tsconfig-д esModuleInterop байхгүйгээс Express import алдаа гарсан — нэмж засав
- partA/lib/ хавтас хоосон байсан — файлуудыг дахин үүсгэж засав

## Дүгнэлт
API дизайны зарчмууд практикт хэр чухал болохыг ойлгов. Factory
паттерн ашиглан хэрэгжилтийг нуух нь кодын уян хатан байдлыг
нэмэгдүүлдгийг сурав. Цаашид алдааны зохицуулалтыг илүү
сайжруулж болох байсан.
